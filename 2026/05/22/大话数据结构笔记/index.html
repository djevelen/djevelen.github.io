chapter 1:什么是数据结构

对于基本概念十分了解的朋友建议直接跳到抽象数据类型那里,把基本的格式看一下就可以进入下一篇blog了.对基本概念还不是很了解的朋友建议耐心看完哦~

数据结构是一门研究非数值计算的程序设计问题中的操作对象,以及它们之间关系和操作等的学科

  • 数据:描述客观事物的符号,是计算机中可以操作的对象;
  • 数据元素:组成数据的、有一定意义的基本单位.(可以将数据理解为一个集合);
  • 数据项:数据不可再分的最小单位.但实际讨论问题时,==数据元素==才是构建数据模型的着眼点
  • 结构:元素之间不是独立的,我们把它们之间的关系就叫做结构.

数据结构又可分为逻辑结构和物理结构

  • 逻辑结构:日常中理解的结构,侧重于描述元素之间的关系.它细分为:

    1. 集合结构(数据元素是平等且==无序==的);
    2. 线性结构(数据元素一对一);
    3. 树形结构(数据元素有一对多);
    4. 图形结构(数据元素有多对多).

  • 物理结构:数据的逻辑结构在计算机中的储存形式.它细分为:

    1. 顺序存储:元素存储在连续的的内存空间中,==储存关系可以反映逻辑关系==;
    2. 链式存储:元素随意存储在内存空间中(可以连续也可以不连续),==位置关系不能反映逻辑关系,必须引入指针==.

抽象数据类型 :

一个具体的数学模型和对应的操作.我们完全可以从实际问题出发,做出最简单的归纳,==而不用先去考虑如何用计算机去实现==(实际上就算是从抽象的本质).这是一个很小却很重要的的点.

前言(牢骚)

  • 实际上最开始学数据结构时我也不理解为什么要引入算法,甚至得出了学校数据结构课像杂种的结论.但现在来看想要搞清楚数据结构的几种类型确实很容易,但是想要真正理解它们的用处是==离不开算法==的.算法也许才是数据结构的精髓和难点.所以请对此不熟悉的朋友们耐心读完这一篇文章,希望你会有所收获.

算法是解决具体问题的求解步骤,在计算机中表现为指令的有限序列

  • 算法的五个特性:
    1. 输入和输出;
    2. 有穷性(==有限==步骤里完成);
    3. 确定性(每个步骤不会产生二义性);
    4. 可行性(也许可以粗鄙地理解为正确性,前者针对==过程==,后者针对==结果==);
  • 算法效率的度量方法:事后统计法(顾名思义就是在计算机上老老实实跑一遍代码)和事前分析估算法(喜闻乐见的时间复杂度和空间复杂度);
  • 函数的渐进增长:对于函数$f(n)$和$g(n)$,很难笼统地说哪个的值一定更大,n在不同范围时结果一般不同(求临界值解个不等式就行了).然而当n足够大时,两个函数谁更大确实确定的,我们说更大的函数渐进增长更快.
    • 实际上就是无穷大量比阶的问题(n取正数且足够大),我们完全可以忽略常数,不论它是单独的一项还是系数,因为不影响最终结果(同阶除外).==这也是时间复杂度化简的理论基础==.

时间复杂度的求解本质就是计算目标代码所有语句执行次数总和的最高阶

    • 内层循环中语句的时间复杂度为$O(1)$,执行次数记为常数$\rm C$;

    • 最后总次数:$f(n) = 1 + \frac {n(n+1)}{2} + \rm C$.由上面阶数化简可知,其最高阶应为 $\rm n^2$,故其时间复杂度记为$O(\rm n^2)$.

  • 常见的时间复杂度所耗费的大小关系:

    $O(1) < O(logn)<O(n)<O(nlogn)<O(n^2)<O(n^3)<O(2^n)<O(n!)<O(n^n)$

  • 最坏情况:对于n个元素,最后一次(花了n次)才找到目标对象.最坏情况运行时间是一种==保障==.除非特定规定,不然一般提到的运行时间都是最坏情况运行时间.

  • 平均情况:对于n个元素,花了$\dfrac {\rm n}{2}$次就找到了目标对象.平均运行时间是所有情况中==最有意义==的,因为它是期望的运行时间.

用空间换时间

  • 空间复杂度:通过计算算法所需要的存储空间实现,其计算公式记作:$S(n) = O(f(n))$,n为问题的规模,$f(n)$是问题所占据存储空间的函数.
  • 若算法执行时所需要的辅助空间相对于输入数据量而言是个常量,则称此算法为==原地工作==,空间复杂度为**$O(1)$**.
  • 复杂度前无限定词时,默认指==时间复杂度==.

chapter 3:线性表

线性表是零个或多个数据元素的有限序列

  • 线性表的数学定义:若将线性表记作$(a_1,……a_i-1,a_i,a_i+1……,a_n)$,则表中$a_i-1$领先于$a_i$,$a_i$领先于$a_i+1$,则称$a_i-1$是$a_i$的直接前驱元素,$a_i+1$是$a_i$的直接后继元素.除了==表头和表尾==,线性表中其他元素均==只有一个==前驱和后继元素.

  • 线性表元素的个数$\rm n$定义为线性表的长度.$\rm n$为$0$时,称为空表.

  • 线性表有==顺序==,强调==一一对应==,元素==有限==
